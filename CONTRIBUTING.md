# Contributing

Thanks for your interest in SSM. This is an early project, so small, focused contributions are easiest to review.

## Good contributions

- Documentation fixes
- Reproduction steps for bugs
- Tests for existing workflows
- Small improvements to identity scenario language
- Clear issue reports with expected and actual behavior

## Local setup

```bash
npm install
npm run verify
```

If `verify` is too heavy for your change, run the closest relevant script and mention that in your PR.

## Pull requests

- Keep PRs focused.
- Explain the problem and the change.
- Include test results.
- Do not commit secrets, real customer evidence, or private identity configuration.
